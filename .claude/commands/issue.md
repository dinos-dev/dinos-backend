GitHub 이슈를 생성하고 브랜치를 자동으로 세팅한다.

## 동작 방식

사용자가 만들고 싶은 기능/수정 사항을 설명하면, 아래 정보를 추론하여 `scripts/issue.sh`를 실행한다.

## 추론 규칙

**type** — 아래 기준으로 판단:
- 새 기능 → `feature`
- 버그 수정 → `bug`
- 긴급 수정 → `hotfix`
- 코드 개선/리팩토링 → `refactor`
- 설정·인프라·문서 등 기타 → `fix`

**branch-suffix** — 영어 kebab-case, 기능을 간결하게 표현 (예: `shared-pins`, `user-profile-update`)

**제목** — 한국어로 핵심 기능을 간결하게 서술

**개발 내용 항목** — 구현해야 할 세부 작업을 bullet로 분리 (최소 1개)

## 실행

추론이 완료되면 아래 명령을 Bash 도구로 실행한다:

```bash
AUTO_BRANCH=true ./scripts/issue.sh <type> <branch-suffix> "<제목>" "<내용1>" ["<내용2>" ...]
```

실행 전에 추론한 값을 사용자에게 보여주고 확인을 받는다.
확인 후 스크립트를 실행하고, 완료되면 이슈 번호와 브랜치명을 알려준다.

## 사용 예시

```
/issue 로그인한 유저의 프로필 수정 기능을 만들고 싶어
/issue 북마크 목록 조회 시 페이지네이션이 안 되는 버그 수정
```
