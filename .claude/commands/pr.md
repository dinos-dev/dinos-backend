현재 브랜치의 작업 내용을 분석하여 GitHub PR을 자동으로 생성한다.

## 동작 방식

1. 현재 브랜치명과 git 히스토리를 읽어 작업 내용을 파악한다
2. 연관 이슈 번호를 브랜치명 또는 커밋 메시지에서 추출한다
3. PR 제목과 작업 내용을 자동으로 구성하여 `scripts/pr.sh`를 실행한다

## 컨텍스트 수집 절차

아래 명령들을 순서대로 실행하여 컨텍스트를 수집한다:

```bash
# 1. 현재 브랜치 확인
git branch --show-current

# 2. main 대비 커밋 목록 확인
git log main..HEAD --oneline

# 3. 변경된 파일 목록 확인
git diff main..HEAD --name-only

# 4. 변경 내용 요약 (너무 크면 --stat만)
git diff main..HEAD --stat
```

## 추론 규칙

**이슈 번호** — 우선순위 순:
1. 커밋 메시지에 `#숫자` 패턴이 있으면 사용
2. 없으면 사용자에게 직접 질문

**PR 제목 suffix** — 브랜치명과 커밋 내용을 종합해 한국어로 간결하게 서술

**작업 내용 항목** — git log와 diff --stat을 분석하여 의미 있는 작업 단위로 bullet 구성

## 실행

추론이 완료되면 아래 명령을 Bash 도구로 실행한다:

```bash
./scripts/pr.sh <issue-number> "<PR 제목>" "<작업내용1>" ["<작업내용2>" ...]
```

실행 전에 구성한 PR 내용을 사용자에게 보여주고 확인을 받는다.
확인 후 스크립트를 실행하고, 완료되면 PR URL을 알려준다.

## 사용 예시

```
/pr
/pr 이슈번호는 85야
```
